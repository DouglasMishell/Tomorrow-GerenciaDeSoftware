    -Gerência e Configuração de Software 

    O Git-SCM, ou simplesmente Git, é um sistema de controle de versão distribuído desenvolvido por Linus Torvalds, o criador do Linux. 
    Ele foi projetado para lidar com projetos de qualquer tamanho, desde pequenos projetos pessoais até grandes projetos. 
    Git-SCM permite que várias pessoas trabalhem  simultaneamente no mesmo projeto, mantendo um histórico completo de todas as alterações feitas.

----------------------------------------------------------Controle de MUdança-----------------------------------------------------------------------------------
    *Issue Tracking
        
        SIstema para averiguar as tarefas e funcionalidades que cada um está responsável; possui nome e descrição.

        *Milestone

        É um agrupamento de issues, associados pelo seu objetivo, como por exemplo: Milestone de cadastro de usuário.

    *Workflows
        
        É a maneira de como sua equipe irá realizar o fluxo de trabalho. Podendo ser dividido em duas formas.

    *Centralizado
            Os devs não criam remificações na branch principal e fazem push para o repositório central.

    *Feature Branches
            OS devs criam ramificações para cada tarefa.Após a conclusão realizam o merge.
    
    *Vantagens do Controle De Mudança 
            Ter uma organização do seu projeto é essencial para o seu desenvolvimento, e através dos conceitos e ferramentas apresentados acima
            seus erros serão mitigados, seu projeto terá maior fluidez de escalonamento e melhores condições de trabalho.
    
-------------------------------------------------------Controle de Versionamento-----------------------------------------------------------------------------------

    -O que é? 

            Obter os registros de todo o seu código é essencial, você terá a possibilidade de ver cada etapa do seu projeto, cada funcionalidade desenvolvida
        e as informações sobre aquela mudança, mantendo o seu projeto mais organizado.

            A tecnologia mais comumente utilizada é o Git, trabalhando lado a lado com o GitHub,plataforma responsável pelo histórico do projeto
        e como repositório(local onde se guarda o código) remoto. A plataforma pode funcionar como uma rede social, onde você terá acesso aos repositórios públicos, podendo trocar informações
        com os respectivos desenvolvedores.


    - Comandos do Git 
        Informações sobre quem está realizando as modificações.
            git config --global user.name "Fulano de Tal" 

            git config --global user.email "fulanoDeTal@email.com"
        Iniciar o repositório local
            git init
        Estado atual do repositório 
            git status
        Adicionando arquivos
            
            O git add coloca o arquivo em um estágio pré commit.
            git add "nome do arquivo"
            git add .       adiciona todos os modificados
            git add *.txt   adiciona todos os arquivos com essa extensão
        Enviando para o repositório
            
            git commit -m
